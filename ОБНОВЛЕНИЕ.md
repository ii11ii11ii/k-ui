# Обновление библиотеки → новый «k»-релиз в npm

Коротко: эта папка `k-ui/` уже содержит готовую версию **0.4.0** с полным
переименованием на префикс **`k`** (`KIcon`, `KDataTable`, классы `.k-*`).
Чтобы у потребителей `npm install @ii11/k-ui` встал новый «k»-код — нужно
залить эти файлы в репозиторий и выпустить релиз. Ниже — три шага.

---

## Шаг 1. Залить новые файлы в GitHub-репозиторий

Репозиторий: **`github.com/ii11ii11ii/k-ui`** (корень репо = эта папка `k-ui/`).

### Вариант А — через сайт (без терминала)
1. Открыть репозиторий → **Add file → Upload files**.
2. Перетащить **содержимое папки `k-ui/`** (не саму папку, а её содержимое:
   `packages/`, `stories/`, `docs/`, `README.md`, `package.json`, и т.д.).
   Папки `node_modules/` и `packages/core/dist/` загружать НЕ нужно.
3. Commit прямо в `main` с сообщением `release: k-ui v0.4.0 (k-префикс)`.

### Вариант Б — через git
```bash
cd k-ui
git add .
git commit -m "release: k-ui v0.4.0 (k-префикс)"
git push origin main
```

---

## Шаг 2. Поднять версию (если 0.4.0 уже была опубликована)

npm **не даёт опубликовать одну и ту же версию дважды**.

- Если в npm ещё **нет** версии `0.4.0` → пропустите этот шаг, публикуйте как есть.
- Если `0.4.0` уже занята → поднимите номер в `packages/core/package.json`
  (поле `"version"`), например на `0.4.1`. На сайте это правится прямо в файле
  через карандаш ✏️ → Commit. Локально: `npm -w packages/core version patch`.

> Проверить, что уже опубликовано: страница пакета на npmjs.com →
> вкладка **Versions**.

---

## Шаг 3. Выпустить релиз → робот опубликует в npm

Публикация автоматическая (workflow `.github/workflows/publish.yml`).
Нужен один раз настроенный секрет **`NPM_TOKEN`** (см. `PUBLISHING.md`, раздел B).

Любой из двух способов:
- **Releases → Create a new release** → тег `v0.4.0` (или `v0.4.1`) → **Publish release**.
- **Actions → Publish @ii11/k-ui → Run workflow**.

Робот соберёт пакет (`npm run build`) и зальёт в публичный npm. Через ~1 минуту:

```bash
npm install @ii11/k-ui@latest   # встанет новый k-релиз
```

## Проверка у потребителя

```ts
// main.ts — глобально
import KometumUI from '@ii11/k-ui'
import '@ii11/k-ui/style.css'
createApp(App).use(KometumUI)
```
```vue
<!-- точечно: имена с префиксом K, теги <k-*> -->
<script setup lang="ts">
import { KDataTable, KFormField, useFormGuard } from '@ii11/k-ui'
</script>
<template>
  <k-data-table … />
</template>
```

⚠️ **Это ломающий релиз для проектов-потребителей.** Старые имена без `k`
(`<icon>`, `{ Icon }`, классы `.kui-*` / `.btn` / `.fld`) больше не работают —
их нужно переименовать. Полный список замен — `docs/CHANGELOG.md`, раздел `0.4.0`.
